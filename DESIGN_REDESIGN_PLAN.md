## Executive Summary

HormonaIQ has the right design foundation — the Morning Garden color system, Instrument Serif typography, phase-aware tokens, and thoughtful clinical copy — but reads as generic because it has:
- **Zero proprietary illustration language** (no custom illustrations, empty states, or onboarding art)
- **A placeholder AI identity** (Ora is a single italic letter "O" — not a mark, not a brand)
- **Generic SVG stroke icons** copied from public libraries (Heroicons, Feather)
- **Inconsistent component craft** (cycle ring is a basic SVG circle, voice note button says "Rec")

All of this is fixable without touching the app's architecture or clinical logic.

**Target feeling**: "This company is looking out for me. I am safe here."  
**Current feeling**: "This was generated by an AI following a template."

## Part 2 — Benchmarks: What Best-in-Class Apps Do

### Findings from: Flo, Clue, Natural Cycles, Headspace, Calm, Bearable, Apple Health, Ovia

| App | Key Differentiator | Apply to HormonaIQ |
|---|---|---|
| **Headspace** | Illustration system IS the brand. Faceless characters convey emotion through posture. | Botanical geometric system must be the brand, not decoration |
| **Clue** | Color is diagnostic, not decorative. Empty states are educational, not apologetic. | Phase color = data signal, not wallpaper |
| **Calm** | Typography does emotional heavy lifting. Warm serif for all emotional moments. | Instrument Serif reserved for emotional moments only (not captions) |
| **Bearable** | Avoids toxic positivity. "Log how you're feeling" not "Track your wellness journey!" | All copy audit: remove aspiration language from sensitive contexts |
| **Ovia** | Milestone moments are ceremonial (full-screen, generous) not gamified (badge + points) | Milestone celebrations use botanical geometry expansion, no confetti |
| **Apple Health** | Premium = white space. Negative space communicates "we are not overwhelming you." | Target <60% screen content occupation on home screen |
| **Natural Cycles** | Trust signals woven inline into narrative, not in footnotes | Clinical credibility signals belong in onboarding flow, not disclaimer text |

### Anti-Generic Rules (Never Do)

1. Never use outline-only icons from Heroicons / Feather / Lucide without significant modification
2. Never use red for symptom scores or period tracking (use deep coral, warm amber)
3. Never stack 3+ equal-weight cards on home screen
4. Never write apologetic empty states ("Nothing here yet!", "No data to show!")
5. Never animate symptom sliders or pain scores (trivializes the experience)
6. Never give Ora a chatbot speech bubble UI
7. Never use uppercase for emotionally sensitive content (SI logging, crisis support)
8. Never use a pure white (#FFFFFF) primary background — always warm cream
9. Never use stock illustrations of women in yoga poses or aspirational wellness imagery
10. Never use gamification mechanics (streaks with flames, level-up badges) for chronic condition tracking

## Part 3 — Resolved Design Direction

After critic debate (pragmatist vs. user advocate vs. brand strategist), the following directions are resolved:

### Illustration System: Botanical Geometry

**Decision**: Soft geometric abstraction with botanical logic — NOT botanical line art

**Botanical line art was rejected because**:
- SVG strokes degrade on older Android at different pixel densities
- Dark mode requires rethinking every asset (fills just swap color token)
- Reads as "Etsy apothecary brand" — already a cliché in women's health (2024-2026)
- Requires visual parsing of detail — cognitively expensive for brain fog users

**Botanical geometry is**: Organic, cellular, growth-derived forms expressed as filled SVG paths. Think leaf cross-sections, seed geometry, petal arrangements — abstracted into geometric forms, not illustrated.

**Applied ONLY to**: Empty states, onboarding hero screens, Ora's interface boundary, milestone moments, phase ring interior fill. NOT to navigation icons or functional UI chrome.

**Style rules**:
- No outlines — filled forms only at illustration scale
- Max 5 colors per illustration (from Morning Garden palette)
- Shapes suggest botanical origin without literal representation
- Rounded, never sharp — all forms convey safety through roundness
- Operates on the eucalyptus/cream/coral/sage palette already defined

---

### Ora Identity: The Botanical Mark

**Decision**: A single continuous closed-form shape derived from cross-sectioned seed geometry. No face, no character, no speech bubble.

**Specification**:
- Size: 32dp primary, 16dp compact (Ora presence indicator)
- Color: `colors.eucalyptus` (#3F6F5A)
- Animation: Breathing opacity loop — 85% → 100% → 85% over 4 seconds (`Animated.loop`)
- Appears ONLY when Ora is active in the interface
- Never appears decoratively
- No "ORA" text label after onboarding — the mark earns recognition through repetition
- In crisis/SI context: mark does NOT appear (Ora steps back, clinical clarity takes over)

**States**:
- `ora-listening`: Full opacity (100%), breathing animation active
- `ora-thinking`: Opacity pulse faster (2s loop) with subtle scale 1.0→1.02
- `ora-insight`: Mark + single botanical leaf-form emerging from its center
- `ora-unavailable`: 20% opacity, no animation

---

### Cycle Ring: Circle Stays, Interior Transforms

**Decision**: Ring shape is permanent and consistent. Investment in what's inside.

**Why morphing was rejected**: Path interpolation in RN SVG requires identical node counts between morph states — a maintenance trap. Medical credibility requires visual stability (same shape = reliable instrument). The Apple Watch rings never morph — that's why they became iconic.

**What changes**:
1. **Phase-aware gradient fill** inside the ring (not on the arc) — sage fill in follicular, butter in ovulatory, coral in luteal, rose in menstrual
2. **Botanical accent mark** at the current day position: a small leaf-form SVG that rotates around the ring as a pointer
3. **Typography refinement**: Larger day number (Instrument Serif 40px), smaller phase label (Inter 10px uppercase, ink3), more contrast between the two
4. **Pulse animation fix**: Scale only the phase-fill interior, not the entire ring + number

---

### Phase Color: Subtle, Stable, Never Darkening

**Decision**: Phase-aware color ONLY in ring interior fill + phase pill component.

- Home screen background holds stable warm cream at all phases
- An "Atmosphere Mode" setting (off by default) adds ≤8% opacity phase tint to screen background
- Phase shifts NEVER make the screen darker — darkness is user-controlled via dark mode only
- Ethical reasoning: Forcing a dark UI on a luteal user tells her "the app expects you to feel heavy" — potentially reductive

---

### Icons: Refined Standard (NOT Botanical)

**Decision**: Navigation and functional icons stay legible with standard metaphors (shield = safety, lock = privacy).

**Refinements applied**:
- Normalize all icons to 1.5px stroke weight
- Rounded linecaps and linejoin on all paths
- Consistent 24dp grid
- Phase-color active/selected state fills
- No fill on resting state — stroke only

**Botanical system handles**: Illustrations, empty states, onboarding — NOT navigation or buttons.

---

### Animation: 250ms Default, Revelational Moments Only at 350ms

| Context | Duration | Easing |
|---|---|---|
| Navigation transitions | 250ms | `Easing.inOut(Easing.ease)` |
| Micro-interactions (tap feedback, toggle) | 150ms | `Easing.out(Easing.ease)` |
| Cycle ring reveal / phase transition | 350ms | Spring physics |
| Ora first appearance / milestone moments | 400ms | Spring physics |
| Reduce Motion enabled | 100ms all | No spring |

Always wires `AccessibilityInfo.isReduceMotionEnabled()` — collapse all animations to 100ms when enabled. The app's existing `useReducedMotion()` from `react-native-reanimated` covers this.
